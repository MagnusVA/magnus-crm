## 🎭 Context: Why Phase 4 Exists

**The Follow-Up Dialog is a state-machine UI** (idle → loading → success | error), not a form. It has:

- **5 `useState` hooks** — all for UI flow (dialog visibility, loading state, result display, clipboard feedback)
- **Zero user-editable fields** — the booking URL is **read-only**, generated by backend
- **No form submission** — the "Generate" button calls a Convex action directly
- **No validation errors** — errors are action-level (network/backend), not field-level

**Conclusion**: This dialog **does not benefit from RHF + Zod migration**. It stays as-is.

**Future**: The v0.5 Phase 4 (Follow-Up & Rescheduling Overhaul) will completely redesign this dialog with a two-path flow:
1. "Send Link" — similar to current (state-machine, no form)
2. "Set Reminder" — **new form with RHF + Zod** (reminder method, date/time, notes)
